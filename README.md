# CarDashboard — цифровая приборная панель

Qt 6.10 · QML 2.15 · C++17 · QPainter · qmake/MinGW 64-bit (Windows) · Kvaser CANlib

Премиальный автомобильный кластер в стиле виртуального кокпита: два круговых прибора
(спидометр / тахометр) и центральная мультиинформационная панель. Данные приходят либо
из встроенного симулятора, либо по шине **CAN** (Kvaser) — источник прозрачно
заменяется, UI к этому уже готов.

---

## Возможности

- **Спидометр** 0–300 км/ч (нелинейная шкала, центр на 100 км/ч) и **тахометр** 0–8000 об/мин
  (линейный, ×1000) с индикатором передачи (P / N / 1–6) в центре.
- **Красная подсветка обода**, плавно нарастающая с оборотами; sweep-анимация стрелок при запуске.
- **Центральная панель** (переключается с анимацией):
  - время / дата по краям, иконки фар, виджет круиз-контроля;
  - **Удержание в полосе (LKAS)** — перспективная дорога с белой разметкой и жёлтым
    полым треугольником-«машиной», лежащим на полосе; включается кнопкой, привязан к CAN;
  - **Мониторинг автомобиля** (когда LKAS выключен) — модель машины вид сверху,
    давление 4 шин, состояние масла / АКБ / охлаждения;
  - мгновенный и средний расход (с цветовыми состояниями), пробег, температура ДВС, запас хода;
  - стильная шкала топлива (градиентная заливка, деления E·¼·½·¾·F, проценты, литры).
- **Индикаторы**: критические (красные, верхний стрип) и предупреждения (жёлтые, нижний стрип),
  всегда в одну строку с авто-подгонкой размера; поворотники у границ приборов; self-check при старте.
- **Панель управления** снизу: запуск двигателя, газ/тормоз/сцепление, ручник, повороты/аварийка,
  свет, круиз, удержание в полосе, тест-кнопки всех индикаторов, подключение CAN.

## Структура проекта

```
car_dashboard/
├── CarDashboard_v7_CL.pro      — проект qmake (TARGET = CarDashboard)
├── resources.qrc               — QML, шрифты, иконки (встроены в бинарник)
├── main.cpp                    — точка входа: регистрация типов и контекст-свойств
├── backend/
│   ├── DataModel.h/.cpp        — Q_PROPERTY-модель данных (приёмник CAN/симулятора)
│   ├── Controller.h/.cpp       — Q_INVOKABLE-команды управления
│   └── Simulator.h/.cpp        — QTimer 33 мс: физика, расход, круиз-PID, турбо-лаг
├── ui/
│   ├── GaugeItem.h/.cpp        — QQuickPaintedItem: рендер прибора через QPainter (FBO + кэш)
│   ├── CANReader.h/.cpp        — чтение Kvaser CANlib в рабочем потоке, декодирование кадров
│   ├── main.qml                — окно 3840×2160 (4K), масштабируется
│   ├── Dashboard.qml           — компоновка кластера и z-слоёв
│   ├── Gauge.qml               — QML-обёртка прибора (sweep, мигание красной зоны)
│   ├── PanelOutline.qml        — хром-контур кластера + RPM-свечение
│   ├── Display.qml             — центральная панель (часы, LKAS/монитор, расход, топливо…)
│   ├── CriticalStrip.qml       — стрип красных индикаторов (одна строка, авто-размер)
│   ├── WarningStrip.qml        — стрип жёлтых индикаторов
│   ├── IndicatorPanel.qml      — логика активных критических надписей для Display
│   ├── ControlsOverlay.qml     — нижняя панель управления
│   ├── BlinkController.qml      — singleton-таймер мигания (поворотники)
│   └── DesignSystem.qml        — singleton с палитрой/константами
└── assets/
    ├── fonts/                  — Microgramma-Normal.ttf (+ Inter как fallback)
    └── icons/                  — PNG-иконки индикаторов и света
```

> В `resources.qrc` присутствуют и устаревшие/незадействованные модули (`Indicator.qml`,
> `TurnSignals.qml`, `HeadlightStrip.qml`, `ControlsWindow.qml`) — оставлены как история,
> в текущей компоновке не используются.

## Архитектура и поток данных

```
            ┌─────────────┐   сигналы   ┌─────────────┐
   CAN  ───▶│  CANReader  │────────────▶│             │
            └─────────────┘             │             │   Q_PROPERTY     ┌──────────┐
                                        │  DataModel  │─────────────────▶│   QML    │
            ┌─────────────┐   пишет     │             │   (NOTIFY)       │  (UI)    │
   демо ───▶│  Simulator  │────────────▶│             │◀─────────────────│          │
            └─────────────┘             └─────────────┘  Controller      └──────────┘
                  ▲                                      (Q_INVOKABLE)         │
                  └──────────────────────────────────────────────────────────┘
```

- **DataModel** — единый источник истины (~45 Q_PROPERTY): `speed`, `rpm`, `fuelLevel`,
  `engineTemp`, `odometer`, `currentGear`, флаги критических/предупреждающих/информационных
  индикаторов, `cruiseActive`, `laneAssist`, давление шин `tirePress FL/FR/RL/RR`, `batteryVoltage`.
- **Controller** — команды из QML (старт/стоп, педали, свет, повороты, круиз, LKAS, тест-индикаторы).
- **Simulator** — тик 33 мс: переключение передач, расход (л/100 и л/ч), PID круиз-контроля,
  турбо-лаг, пробуксовка; пишет результат в DataModel.
- **CANReader** — при подключении Kvaser декодирует кадры (таблица `CanID` в `CANReader.h`)
  и пишет в тот же DataModel, заменяя симулятор.

## Сборка (Windows, MinGW 64-bit)

**Qt Creator**
1. Открыть `CarDashboard_v7_CL.pro`.
2. Набор: **Desktop Qt 6.10.x MinGW 64-bit**.
3. Run.

**Командная строка** (пример путей этого проекта)
```bat
set PATH=C:\Qt\Tools\mingw1310_64\bin;C:\Qt\6.10.2\mingw_64\bin;%PATH%
cd build\Desktop_Qt_6_10_2_MinGW_64_bit-Debug
qmake ..\..\CarDashboard_v7_CL.pro
mingw32-make -j8
```

### Сборка на macOS / Linux (без CAN)

CAN-драйвер Kvaser есть только под Windows, поэтому на macOS/Linux проект собирается
**без CAN** — `CANReader` использует встроенную заглушку, кнопка «CAN» корректно
сообщает «не подключён», данные даёт симулятор. Ничего отключать вручную не нужно.

**Qt Creator (macOS):** открыть `CarDashboard_v7_CL.pro` → набор **Desktop Qt 6.x (clang)** → Run.

**Командная строка:**
```bash
mkdir -p build && cd build
qmake ../CarDashboard_v7_CL.pro
make -j8
./CarDashboard.app/Contents/MacOS/CarDashboard   # macOS
# ./CarDashboard                                  # Linux
```

При сборке qmake печатает `Платформа без Kvaser — сборка БЕЗ CAN (заглушка)`.
Архитектура (arm64 / x86_64) берётся из выбранного Qt-кита — работает и на Apple Silicon, и на Intel.

> **Важно про QML.** Файлы `.qml` встроены в `resources.qrc`, поэтому после их правки
> нужно перезапустить `rcc` — иначе пересоберётся только C++, а интерфейс останется старым.
> Самый простой способ из консоли: обновить время файла ресурсов перед сборкой
> (`touch resources.qrc` / `(Get-Item resources.qrc).LastWriteTime = Get-Date`).
> Qt Creator делает это сам при обычной сборке.

## CAN-интеграция (Kvaser)

- Драйвер: **Kvaser CANlib** (`C:/Program Files (x86)/Kvaser/Canlib`), путь прописан в `.pro`.
- Подключение/отключение — кнопка **CAN** в панели управления; канал и битрейт (по умолчанию
  500 kbit/s) задаются свойствами `CANReader`.
- Карта кадров (11-bit) задаётся в `ui/CANReader.h` (`namespace CanID`): `SPEED`, `RPM`,
  `ENGINE_TEMP`, `FUEL_LEVEL`, `GEAR`, битовые `STATUS_1/2` для индикаторов. Подгоняется под конкретный ECU.
- Чтение идёт в отдельном `QThread` (poll 10 мс), декодированные значения уходят в `DataModel`.

Без железа проект собирается и работает на симуляторе (на macOS/Linux CANlib — заглушка).

## Оптимизация (важно для CAN/Android)

- **Кэш статического слоя прибора**: фон, деления и подписи рисуются один раз в `QImage`
  и переиспользуются; каждый кадр поверх рисуются только дуга, свечение, стрелка, текст.
- **Throttle перерисовки** RPM-свечения контура (`PanelOutline`) — по заметной дельте, не 60 fps.
- Шкала топлива, ЭКО/монитор и анимации режимов — обычные `Item`/`Rectangle` (GPU-батчинг),
  Canvas задействован минимально и перерисовывается только когда нужен (например, дорога LKAS — лишь когда режим активен).
- Стрипы индикаторов — одна строка с авто-подгонкой (нет лишних перерисовок и переноса).
- Данные идут только через `DataModel` — подключение CAN не требует изменений UI.

> Для продакшена собирать в **Release** (без `QT_QML_DEBUG`, с оптимизациями компилятора).

## Нюансы

- Шрифт **Microgramma** — латиница; кириллица и символ `°` берутся из шрифта-fallback
  (в коде температура рисуется числом Microgramma + «°C» дефолтным шрифтом).
- Окно задано как 4K (3840×2160) и масштабируется под фактическое разрешение экрана.
